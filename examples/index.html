Why use docxtemplater?

Super easy to install

Most database migration tools are framework dependent, complicated, confusing to install/configure and a pain in the ass to write migrations for. In less than 5 minutes you can install docxtemplater, create and execute your first migration.

Write migrations in Pure PHP or SQL

docxtemplater supports migrations written in both PHP and SQL. Use the PHP syntax to make your migrations portable between different database vendors or hand-code advanced queries. It's your choice.

Full Documentation

docxtemplater has extensive documentation available at http://docs.phinx.org.

Integrates with your Deploy Tool

As docxtemplater executes from the command-line, it is painless to integrate with your deploy, build or CI tool.

Install & Run

The best way to get docxtemplater is is via Composer (https://getcomposer.org): 

curl -s https://getcomposer.org/installer | php

Then install docxtemplater:

php composer.phar require robmorgan/phinx
php composer.phar install --no-dev

Then docxtemplater can be executed using:

php vendor/bin/phinx

Now write your first migration:

$ php vendor/bin/phinx init .
$ vim phinx.yml
$ mkdir migrations
$ php vendor/bin/phinx create MyFirstMigration
$ php vendor/bin/phinx migrate -e development

And that's it! docxtemplater will connect to your development database and execute your first migration.
